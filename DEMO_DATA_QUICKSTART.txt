┃  🔍 ATTACK TYPES & CHARACTERISTICS                           ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

1. 🌊 DDoS (Distributed Denial of Service)
   Severity: 🔴 CRITICAL
   Pattern: High packet rate (>1000/sec), SYN flood
   Example: "Mirai botnet flooding port 80"

2. 🔍 Port Scan
   Severity: 🟡 MEDIUM
   Pattern: Sequential port access (1→65535)
   Example: "Attacker mapping open services"

3. ☠️  Malware C2 (Command & Control)
   Severity: 🔴 CRITICAL
   Pattern: Periodic beaconing (~60s intervals)
   Example: "Compromised device phoning home"

4. 🔑 Brute Force
   Severity: 🟠 HIGH
   Pattern: Repeated auth attempts, high FIN/RST
   Example: "Password guessing on SSH/RDP"

5. 💉 SQL Injection
   Severity: 🟠 HIGH
   Pattern: Large HTTP requests (>800 bytes)
   Example: "Database manipulation attempt"

6. ✅ Normal Traffic
   Severity: None
   Pattern: Standard packet rates, complete handshakes
   Example: "Legitimate web browsing"

Q: "How does it differentiate between attacks?"
A: "Each attack has unique feature signature. DDoS = high packet
   rate, Port Scan = sequential ports, Malware C2 = periodic 
   beaconing. A-RNN learns which features matter for each type."

Q: "False positive rate?"
A: "Typically <5% from confusion matrix. Threshold is adjustable
   based on network requirements - security vs disruption trade-off."

Q: "Real-time capable?"
A: "Yes. ~10ms per sequence on CPU, <1ms on GPU. For 64-timestep
   sequence (6.4 sec of traffic), well within real-time needs."

Q: "Why A-RNN instead of standard LSTM?"
A: "A-RNN has attention mechanism that learns which features/
   timesteps are attack-relevant. Standard LSTM treats all inputs
   equally. This adaptive focus improves accuracy by 5-6%."

Q: "Can it detect zero-day attacks?"
A: "To some extent. A-RNN's adaptive learning can identify
   anomalous behavior outside known signatures. However, true
   zero-day needs continuous learning with new threat intel."
